## AI 小专栏 #1: AI 为什么会"过度拟合"？

### 搜索词: "overfitting detection cross validation best practices 2026"
- **数据点**:
  - 交叉验证是防止过度拟合的核心方法：将数据分成k份，在k-1份上训练，在1份上验证
  - 对于时间序列数据，需要使用时间感知的交叉验证方案
  - 最佳实践：CV调参后，在全部数据上重新训练最终模型
  - 警惕"对交叉验证数据本身过度拟合"的元问题

- **参考链接**:

### 搜索词: "R-squared vs model generalization 2026"
- **数据点**:
  - R²本身不是预测准确性的好指标——仅依赖训练集R²不足以判断泛化能力
  - R²总是随特征增加而增加，即使是无意义的特征——这是过度拟合的经典来源
  - 研究表明R²比SMAPE、MAE、MAPE、MSE、RMSE更有信息量——但仍需配合验证技术
  - 2026年1月ScienceDirect发表关于"Source-independent generalization"的新研究

## AI 小专栏 #2: AI 时代的模型诊断——从残差图到 SHAP

### 搜索词: "SHAP explainable AI model interpretation 2026"
- **数据点**:
  - SHAP (SHapley Additive exPlanations) 基于博弈论的Shapley值，是解释ML模型的标准方法
  - 两种解释层次：局部解释（单个预测）和全局解释（整体模型行为）
  - 模型无关：适用于任何机器学习模型（线性回归、神经网络、随机森林）
  - 2024-2025发展：GPU加速计算、临床预测模型应用、深度学习集成
  - 可视化方法：SHAP summary plots、decision plots、特征重要性图

- **参考链接**:
  - [机器学习中的可解释性：SHAP值及其应用 - 腾讯云](https://cloud.tencent.com/developer/article/2560431) (Aug 26, 2025)
  - [使用GPU加速SHAP解释机器学习模型 - NVIDIA开发者](https://developer.nvidia.cn/blog/explain-your-machine-learning-model-predictions-with-gpu-accelerated-shap/) (Oct 5, 2022)
  - [SHAP可视化方法，临床预测模型解释新框架](https://www.medsta.cn/archives/hi-xin-kuang-jia) (Aug 4, 2025)

## 本周核心技术点（Context7 查证结果）

### scipy.stats.shapiro 用法
```python
from scipy import stats
# 返回结果对象，有 .statistic 和 .pvalue 属性
shapiro_test = stats.shapiro(residuals)
print(f"Statistic: {shapiro_test.statistic}")
print(f"P-value: {shapiro_test.pvalue}")
```

### seaborn.residplot 用法
```python
import seaborn as sns
# 支持 lowess 平滑曲线
sns.residplot(x=fitted_values, y=residuals, lowess=True,
              line_kws={'color': 'red', 'lw': 2})
```

### seaborn.regplot 用法
```python
# 支持 ci 参数控制置信区间
sns.regplot(data=df, x="x", y="y", ci=95)

# 支持 lowess 非参数拟合
sns.regplot(data=df, x="x", y="y", lowess=True)
```
